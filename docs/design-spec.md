# UI 设计规范 - Product Eye（老年友好版）

## 设计原则
本应用的核心用户是老年人，所有设计决策优先考虑可读性和操作简易性。

1. **一眼看清**：文字足够大，对比度足够高
2. **一按就准**：按钮足够大，间距足够宽
3. **一步一事**：每个页面只做一个操作，避免选择困难
4. **双重反馈**：视觉 + 语音同步反馈，确保信息传达

## 布局规范

- 单栏布局，最大宽度 480px，居中
- 内容区左右留白 16px
- 页面顶部统一：返回按钮 + 页面标题
- 主操作区居中，占据视觉重心
- 底部留白不少于 24px

## 交互规范

### 禁止的交互
- ❌ 滑动手势
- ❌ 长按操作
- ❌ 双击
- ❌ 下拉刷新
- ❌ 捏合缩放
- ❌ 弹窗/模态框（用全屏页面替代）

### 允许的交互
- ✅ 单击按钮
- ✅ 滚动列表（仅商品列表页）
- ✅ 输入文字

### 反馈规范
- 按钮按下：背景色加深 + 轻微缩放（< 0.1s）
- 识别中：大号加载文字 + 简单动画（如"识别中..."三点闪烁）
- 识别成功：绿色大号结果 + 自动语音播报
- 识别失败：橙色提示 + 语音播报"未识别出商品，请重试"

## 语音反馈规范
- 识别成功："[商品名称]。价格：[数字] 元。"
- 识别失败："未识别出商品，请重试。"
- 条码未匹配："未找到此条码对应的商品。"
- 相机错误："无法打开相机。请在手机设置中允许 Chrome 使用相机。"

## 页面清单

| 页面 | 路由 | 核心元素 |
|------|------|------|
| 首页 | `#home` | 四宫格按钮 |
| 条码扫描 | `#scan` | 全屏相机 + 取景框 + 取消按钮 |
| 拍照识别 | `#identify` | 相机预览 + 拍照按钮 |
| 识别结果 | `#result/:id` | 商品图 + 名称 + 价格（大字）+ 返回按钮 |
| 商品列表 | `#products` | 搜索框 + 滚动列表 + 添加按钮 |
| 添加商品 | `#add-product` | 表单 + 拍照录入参考图 + 保存按钮 |
| 编辑商品 | `#edit-product/:id` | 同添加，预填数据 + 删除按钮 |
| 设置 | `#settings` | 语音开关、语速、灵敏度调节 |
