# 产品需求文档 - Product Eye

## 项目定位
一款为老年人设计的 Android 手机应用，通过拍照或扫条码识别商品，自动语音播报用户自定义价格。私人使用，不上架应用商店。

## 目标用户
- 主要用户：老年人（视力可能不佳，不熟悉复杂操作）
- 维护者：用户家属（负责录入商品和价格）

## 功能需求

### F1 - 条码扫描识别
- 打开相机，对准商品条码
- 自动识别条码数字
- 在本地数据库中查找对应商品
- 找到后显示商品名称和价格，语音播报
- 未找到时提示"未找到商品，是否录入？"

### F2 - 拍照外观识别
- 对商品拍照
- 与已录入的参考照片进行图像比对
- 匹配到最相似的商品
- 显示商品名称、价格、匹配置信度
- 语音播报结果

### F3 - 商品管理
- 添加商品：拍照（参考图）、输入名称、输入价格、可选扫码录入条码
- 每个商品可存储多张参考照片（建议 3-5 张，不同角度）
- 商品列表：按名称搜索、浏览
- 编辑商品：修改名称、价格、条码、增删参考图
- 删除商品：确认后删除

### F4 - 语音播报
- 识别成功后自动播放："[商品名称]。价格：[金额] 元。"
- 识别失败时播放："未识别出商品，请重试。"
- 支持调整语速（设置中）

### F5 - 离线运行
- 首次加载后，所有功能不依赖网络
- 所有数据（商品信息、参考图、模型）存储在手机本地

### F6 - 设置
- 语音播报开关
- 语速调节（慢/正常/快）
- 音量调节
- 匹配灵敏度调节

## 非功能需求

### N1 - 老年友好
- 字体大（最小 20px）
- 按钮大（最小 60×60px）
- 高对比度配色
- 每页只有一个主要操作
- 无复杂手势（无滑动、无长按、无双击）

### N2 - 隐私安全
- 所有数据仅存储在手机本地
- 不上传任何数据到服务器
- 不需要任何账号登录

### N3 - 可用性
- 应用响应时间 < 2 秒
- 条码识别 < 3 秒
- 拍照识别 < 5 秒
- 支持 10-50 种商品

### N4 - 兼容性
- 最低支持 Android 10 + Chrome 浏览器
- 横竖屏均可用（优先竖屏）
- 适配不同屏幕尺寸
